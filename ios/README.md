# RoofTrace iOS capture app

A thin native "smart camera" (per ADR-007): it captures the richest sensor
bundle a Pro iPhone can produce — RGB photos, per-frame `sceneDepth` maps, the
accumulated ARKit world mesh, GPS fixes, and per-photo attitude — and uploads it
as one multipart POST. **No on-device measurement or reconstruction**; all
fusion math happens server-side. **No live AR overlays** in the capture flow.

## Requirements

- **Device:** iPhone 12 Pro or newer (any Pro model) — a LiDAR scanner is
  required. The approved demo/test device is an **iPhone 15 Pro**. Non-LiDAR
  devices are detected at the setup check and shown a clear error.
- **OS:** iOS 17.0+.
- **Xcode:** 15+ (CI builds on Xcode 16 / macOS 14). Apple SDKs only — no SPM or
  CocoaPods dependencies (ARKit, CoreLocation, AVFoundation, ModelIO, Metal,
  Compression).

### Regenerating the Xcode project

The `.xcodeproj` is produced deterministically from the on-disk sources so it
never drifts. After adding/removing a Swift file, update the lists in
`gen_pbxproj.py` and run:

```bash
cd ios && python3 gen_pbxproj.py
```

## Backend URL configuration

`AppConfig.backendURL` is read from `Info.plist` key `BackendURL`, which is
populated from the build's `.xcconfig` `BACKEND_URL`:

- **Debug:** `http://localhost:3000`
- **Release:** `https://rooftrace.biograph.dev` (https is enforced in non-Debug
  builds; a missing or non-https value `fatalError`s at launch — the iOS mirror
  of the Rails fail-fast-at-boot convention).

Upload endpoint: `POST <BackendURL>/api/v1/capture-sessions/<job_id>` with
`Authorization: Bearer <capture_token>`.

## Capture flow

1. **Token entry** — paste the 32-char base58 `capture_token` + job UUID, or open
   the deep link `rooftrace://capture?token=...&job_id=...` to pre-fill both.
2. **Setup check** — "point at a wall ~1 m away"; probes ARKit `sceneDepth`.
   Non-LiDAR devices land on a clear "requires an iPhone Pro / iPad Pro with
   LiDAR" terminal screen.
3. **8-prompt walk-around** — 4 corners + 4 facades, each a static 2D
   illustration + instruction + compass-bearing hint + "Tap when ready". Each tap
   captures a photo, the depth map, the camera pose, the attitude quaternion, and
   a GPS fix.
4. **Upload** — the world mesh is exported to OBJ, then the whole bundle posts as
   one 18-part multipart body. Progress bar; on success a tap-to-copy results
   URL; on failure one automatic retry, then Retry / Save-locally.

## The capture bundle (session.json)

The manifest conforms to the frozen contract `shared/ios_session_schema.json`
(`manifest_version` `1.0.0`). Key invariants (see the ADR-007 amendment):

- **GPS altitude is HAE** (`CLLocation.ellipsoidalAltitude`), never MSL.
- **Depth maps are 16-bit grayscale PNG, uint16 millimeters** (`depth_scale`
  1000.0; `0`=no depth, `65535`=65.535 m clamped).
- **Camera matrices are row-major with an explicit transpose** — ARKit's
  `simd_float4x4`/`simd_float3x3` are column-major, so `MatrixSerializer` builds
  row `i` = `[col0[i], col1[i], col2[i], col3[i]]`. Never flatten the matrix.
- **Mesh is one Wavefront OBJ** in `arkit_session_local` coordinates.
- **Multipart wire format:** 18 named parts — `session_json` (application/json),
  `photo_00`…`photo_07` (image/jpeg), `depth_00`…`depth_07` (image/png),
  `world_mesh` (model/obj).

## Building and testing

```bash
cd ios
# build for the simulator
xcodebuild -scheme RoofTrace -sdk iphonesimulator \
  -destination 'platform=iOS Simulator,name=iPhone 16 Pro' build
# unit tests (no device hardware needed — protocol injection + synthetic data)
xcodebuild test -scheme RoofTrace \
  -destination 'platform=iOS Simulator,name=iPhone 16 Pro'
```

The full unit suite runs on the simulator: ARKit-dependent services
(`ARKitSessionManager`, `MeshExporter`) are reached only through the
`CaptureSensing` protocol, so the tests never require a physical LiDAR device.
CI runs the same on a GitHub Actions macOS runner (`.github/workflows/ios.yml`),
plus a Python `jsonschema` validation of the synthetic fixture against the shared
schema.

## Distribution (TestFlight / direct install)

- A **paid Apple Developer account** is approved for TestFlight distribution to
  the iPhone 15 Pro test device. Archive the Release scheme, upload via Xcode
  Organizer / `xcrun altool`, distribute through TestFlight.
- **Fallback:** a **free Apple ID** can direct-install to a single device from
  Xcode (7-day provisioning) when TestFlight isn't available. Set your team in
  the target's Signing & Capabilities; `CODE_SIGN_STYLE = Automatic`.
- A provisioning profile for the test device is required either way.
